# The Founders Korea 개발 로드맵

## Phase 1: 미디어 플랫폼 기반 구축

**목표**: 4대 카테고리 기반 미디어 사이트로 전환

| 순서 | 작업 | 설명 | 우선순위 |
|------|------|------|----------|
| 1-1 | 프로젝트 구조 리팩토링 | `app/` 디렉토리 기반 모듈화 | 높음 |
| 1-2 | DB 도입 | SQLAlchemy + Alembic 셋업, 모델 정의 | 높음 |
| 1-3 | 4대 카테고리 라우트 | Insights, Voice, Weekly, Solution 각각 목록+상세 | 높음 |
| 1-4 | 메인 페이지 개편 | 히어로 + 카테고리별 최신 콘텐츠 미리보기 | 높음 |
| 1-5 | 헤더/네비게이션 개편 | 4대 카테고리 메뉴 + 모바일 햄버거 | 높음 |
| 1-6 | 슬러그 기반 URL | SEO 친화적 URL | 높음 |
| 1-7 | 마크다운 렌더링 | 본문 마크다운 지원 + 코드 하이라이팅 | 높음 |
| 1-8 | SEO 메타 태그 | OG, JSON-LD, sitemap.xml, robots.txt | 높음 |
| 1-9 | 카테고리별 컬러 시스템 | 배지, 카드 보더 등에 카테고리 색상 적용 | 중간 |
| 1-10 | RSS 피드 | `/feed.xml` | 중간 |
| 1-11 | 페이지네이션 | 목록 페이징 | 중간 |

**산출물**: 4개 카테고리를 갖춘 미디어 사이트 기본 틀

---

## Phase 2: CMS & 콘텐츠 수집 시스템

**목표**: 비개발자 콘텐츠 관리 + 시민기자 투고 + 뉴스 자동 수집

| 순서 | 작업 | 설명 | 우선순위 |
|------|------|------|----------|
| 2-1 | 관리자 인증 | 비밀번호 기반 로그인/로그아웃 | 높음 |
| 2-2 | 관리자 대시보드 | 포스트 CRUD, 카테고리별 관리 | 높음 |
| 2-3 | 마크다운 에디터 | 실시간 미리보기, 카테고리/태그 선택 | 높음 |
| 2-4 | Founder's Voice 투고 폼 | 공개 투고 + 관리자 검토/승인 워크플로 | 높음 |
| 2-5 | Weekly Curation 스크래퍼 | 뉴스 자동 수집 (BeautifulSoup) | 중간 |
| 2-6 | 큐레이션 관리 | 수집 뉴스 선별 → 주간 리포트 발행 | 중간 |
| 2-7 | 검색 기능 | 제목/본문 전문 검색 + 카테고리 필터 | 중간 |
| 2-8 | 태그 시스템 | 다중 태그, 태그별 필터 | 중간 |
| 2-9 | 뉴스레터 구독 | 이메일 수집, 구독/해지 | 중간 |
| 2-10 | 소셜 공유 | 카카오, X, LinkedIn, 링크 복사 | 낮음 |
| 2-11 | PostgreSQL 전환 | 운영 DB를 Railway PostgreSQL로 | 중간 |

**산출물**: 완전한 CMS + 시민기자 시스템 + 뉴스 자동 수집

---

## Phase 3: 리드 퍼널 & 커뮤니티

**목표**: 콘텐츠 → 상담 전환 퍼널 완성 + 커뮤니티 기능

| 순서 | 작업 | 설명 | 우선순위 |
|------|------|------|----------|
| 3-1 | 상담 신청 페이지 | `/contact` 리드 폼 + 이메일 알림 | 높음 |
| 3-2 | Success Solution CTA | 케이스 스터디 → 상담 전환 버튼 | 중간 |
| 3-3 | 뉴스레터 자동 발송 | Weekly Curation → 이메일 연동 | 중간 |
| 3-4 | 댓글 시스템 | Giscus 또는 자체 구현 | 낮음 |
| 3-5 | 시민기자 프로필 | 투고 이력, 사업장 소개 | 낮음 |
| 3-6 | 라이트/다크 모드 토글 | 사용자 테마 선택 | 낮음 |
| 3-7 | 분석 대시보드 | 방문자, 인기 콘텐츠, 전환율 | 낮음 |

**산출물**: 리드 퍼널 완성, 커뮤니티 기능

---

## 마일스톤 요약

```
현재         Phase 1           Phase 2              Phase 3
MVP    →    미디어 플랫폼   →   CMS & 수집 시스템  →  리드 퍼널 & 커뮤니티
(샘플 3개)  (4대 카테고리)     (투고, 스크래핑)      (상담 전환, 분석)
                                    ↓
                           한국창업경영센터
                          컨설팅 리드 창출
```

## 즉시 실행 가능한 다음 단계

1. `app/` 디렉토리 구조 생성 및 코드 모듈화
2. SQLAlchemy 모델 정의 (Post, Category, Tag)
3. 4대 카테고리 라우트 구현
4. 메인 페이지 & 헤더 네비게이션 개편
